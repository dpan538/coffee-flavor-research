# Release Checklist

Use this before creating the first formal GitHub release or tag.

1. Confirm the author display name and citation identity.
2. Complete the asset license audit.
3. Confirm all tests, smoke tests, type checks, formatting checks, and
   production build.
4. Set a version number.
5. Update `CITATION.cff`.
6. Add `date-released` only when the release is actually created.
7. Create the GitHub tag.
8. Create the GitHub Release.
9. Optionally connect Zenodo.
10. After a DOI exists, update `CITATION.cff` and `README.md`.
11. Save the release's data snapshot, including descriptor records and source
    registry state.
